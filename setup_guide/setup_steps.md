## 1. Overview of Platforms & Integrations

Num Wallet relies on standard, industry-grade APIs to orchestrate seamless user authentication and multi-chain transaction executions:

| Platform | Role in Num Wallet | Authentication Protocol |
| :--- | :--- | :--- |
| **Twilio Verify** | Global SMS OTP gateway confirming phone number ownership. | REST API Basic Auth |
| **Supabase DB** | Direct client registry mapping phone numbers to multi-chain addresses. | Postgres SSL Client |
| **Privy** | Secure Multi-Party Computation (MPC) embedded wallets. | React Native Provider SDK |
| **LI.FI REST APIs** | Production swap and cross-chain routing engines. | Public API Endpoints |
| **Helius (Solana)** | Dynamic, high-throughput RPC node broadcasting Solana mainnet transactions. | Private HTTP Endpoint |
| **QuickNode (EVM)** | Direct Ethereum/Base mainnet RPC node broadcasting EVM transactions. | Private HTTP Endpoint |
| **Resend** | Background platform email engine for PIN-locked backups. | REST API Bearer Token |

## 2. Step-by-Step Setup Instructions

---

### A. Twilio (International SMS Verification Gateway)
Used to send and verify real 6-digit SMS OTP codes to any mobile number globally upon sign-up or login.

1. **Create an Account:**
   - Sign up at [twilio.com](https://www.twilio.com/) and verify your email.
2. **Access the Console Dashboard:**
   - Locate your **Account SID** and **Auth Token** displayed in the homepage Account Info section.
3. **Configure Twilio Verify:**
   - Search for **Verify** in the sidebar search console.
   - Tap **Services** under the Verify menu dropdown.
   - Click **Create Service**. Name it `Num Wallet Auth`.
   - Set the OTP length to `6` digits.
   - Once created, copy the **Service SID** (starts with `VA...`).
4. **Paste Credentials:**
   - Input your Account SID, Auth Token, and Verify Service SID into `app/constants/config.ts`.

---

### B. Supabase (Unified Account & Registries Database)
Stores user profiles, display names, and maps their 10-digit phone-derived account numbers deterministically to EVM and Solana addresses.

1. **Sign Up & Log In:**
   - Access [supabase.com](https://supabase.com/) and register using GitHub or your email.
2. **Initialize Project:**
   - Click **New Project** in your dashboard.
   - Project Name: `num-wallet`
   - Database Password: `y6evZ7IPMrG2aEi9` (or your custom secure password).
   - Region: Select closest to your primary target market (e.g. Europe West or South America).
3. **Execute Database Registry Table Script:**
   - Tap the **SQL Editor** tab in the Supabase sidebar.
   - Click **New Query**.
   - Copy and paste the following SQL schema to create the live registries mapping table:
     ```sql
     -- Create the live Registries table mapping numbers to addresses
     create table public.registries (
       id bigint generated by default as identity primary key,
       account_number text not null unique,
       name text not null,
       solana_address text not null,
       evm_address text not null,
       ton_address text,
       xrp_address text,
       created_at timestamp with time zone default timezone('utc'::text, now()) not null
     );

     -- Disable Row Level Security (RLS) for anonymous signups
     alter table public.registries disable row level security;
     ```
   - Click **Run**. Verify it returns a `Success` message.
4. **Configure Profile Image Storage Bucket & Schema Extension:**
   - Since we are adding live profile image uploading, you need to create the storage bucket and add the `avatar_url` column to support the uploads:
   - Tap **SQL Editor** in the Supabase sidebar.
   - Click **New Query** and execute the following SQL command:
     ```sql
     -- 1. Append the avatar_url column to the registries table
     ALTER TABLE public.registries ADD COLUMN IF NOT EXISTS avatar_url text;

     -- 2. Create the public bucket named "avatars" for profile photo uploads
     insert into storage.buckets (id, name, public) 
     values ('avatars', 'avatars', true)
     on conflict (id) do nothing;

     -- 3. Create policies to allow public/anonymous uploads to the avatars bucket
     create policy "Allow public uploads" on storage.objects 
     for insert with check (bucket_id = 'avatars');

     -- 4. Create policies to allow public read access to the avatars bucket
     create policy "Allow public read" on storage.objects 
     for select using (bucket_id = 'avatars');
     ```
   - Click **Run** and verify it executes successfully.
5. **Retrieve API Parameters:**
   - Go to **Project Settings** -> **API**.
   - Copy the **Project URL** (e.g., `https://giznbbrfbnsxflfsmefr.supabase.co`).
   - Copy the **anon public API key** (a long JWT string).
6. **Update Constants:**
   - Paste these parameters directly into the Supabase config section in `app/constants/config.ts`.

### C. Privy (Secure MPC Key Management & Embedded Wallets)
Provides embedded Multi-Party Computation key containment, securing seed phrase splits on-device and executing signed blockchain triggers without storing complete keys.

1. **Access Developer Console:**
   - Create an account at [console.privy.io](https://console.privy.io/) and log in.
2. **Create New Project:**
   - Tap **Create Project** and name it `Num Wallet`.
3. **Enable Embedded Wallets:**
   - Go to **Embedded Wallets** under settings.
   - Toggle **Enable Embedded Wallets** to `ON`.
   - Set configuration permissions (e.g. bypass security popups if you want a seamless Web2 login feel).
4. **Acquire Keys:**
   - Navigate to **Project Settings**.
   - Copy the **App ID** (starts with `cl...`).
   - Copy the **Client ID**.
5. **Paste Configurations:**
   - Update `PRIVY_APP_ID` and `PRIVY_CLIENT_ID` in `app/constants/config.ts`.

---

### D. Helius / QuickNode (Solana & EVM Mainnet RPC Providers)
Private, secure RPC node endpoints that bypass congested public channels to broadcast live Mainnet transactions instantly.

1. **Solana Helius Node:**
   - Sign up at [helius.dev](https://www.helius.dev/) using a Solana wallet or Google.
   - Generate a free RPC endpoint for Solana Mainnet.
   - Copy your private HTTP RPC URL (e.g., `https://mainnet.helius-rpc.com/?api-key=YOUR-KEY`).
2. **EVM QuickNode/Ankr Node:**
   - Go to [quicknode.com](https://www.quicknode.com/) or [ankr.com](https://www.ankr.com/).
   - Generate an HTTP RPC endpoint for Ethereum Mainnet.
   - Copy your private EVM HTTP RPC URL.
3. **Update Constants:**
    - Paste these into `SOLANA_RPC_URL` and `EVM_RPC_URL` inside `app/constants/config.ts`.

---

### E. Resend (Automated Platform Email Engine)
Used to send the secure, AES-256 PIN-locked HTML recovery document to users' backup emails programmatically in the background without opening mail composer drafts.

1. **Create an Account:**
   - Sign up at [resend.com](https://resend.com/).
2. **Retrieve API Key:**
   - Go to your dashboard and select **API Keys** in the sidebar.
   - Click **Create API Key**. Name it `Num Wallet Email`.
   - Copy the key (starts with `re_`).
3. **Verify Domain (Optional but Recommended for Production):**
   - Go to **Domains** and verify your domain (e.g. `numwallet.app`) to send emails from your own domain (e.g., `security@numwallet.app`).
4. **Paste Credentials:**
   - Paste your API Key into `RESEND_API_KEY` in `app/constants/config.ts`.
   - If sending from a custom verified domain, change `EMAIL_FROM_ADDRESS` in `config.ts` to your custom address.
